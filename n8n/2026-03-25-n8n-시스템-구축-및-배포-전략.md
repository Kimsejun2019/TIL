### Context
개인 워크플로우 자동화를 위해 n8n 시스템을 구축하는 과정에서 로컬 환경과 클라우드 환경(GCP)에서의 배포 및 운영 방법을 학습했습니다. 특히, 로컬에서 개발된 워크플로우 설정을 클라우드 환경으로 이전하고, GCP의 Always Free 티어를 활용하여 비용 효율적인 n8n 인스턴스를 지속적으로 운영하기 위한 전략을 모색하는 것이 주요 목표였습니다.

### Core
#### n8n 로컬 구동 (n8n Local Execution)
n8n은 Docker를 사용하여 로컬 환경에서 쉽게 구동할 수 있습니다. `docker-compose.yml` 파일을 통해 n8n 컨테이너를 실행하고 포트 매핑을 설정하여 웹 인터페이스에 접근합니다.

'''python
version: '3.8'

services:
  n8n:
    image: n8n.io/n8n
    restart: always
    ports:
      - "5678:5678"
    volumes:
      - ~/.n8n:/home/node/.n8n
    environment:
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
'''

#### n8n GCP 구동 및 로컬 셋팅 값 저장 및 불러오기 (n8n GCP Execution & Configuration Management)
GCP Compute Engine 인스턴스에 Docker를 설치한 후 위와 유사한 `docker-compose.yml` 파일을 사용하여 n8n을 배포합니다. n8n 워크플로우와 자격 증명(credentials)은 웹 UI를 통해 JSON 형식으로 내보내기(export) 및 가져오기(import) 기능을 사용하여 로컬과 클라우드 환경 간에 손쉽게 공유할 수 있습니다. 이는 개발 환경과 운영 환경 간의 일관성을 유지하는 데 중요합니다.

#### GCP Always Free Tier + Docker + Ngrok 조합 구성
GCP Always Free 티어의 Compute Engine 인스턴스를 활용하여 n8n을 배포하고, 이를 Docker 컨테이너로 관리하여 환경의 격리성과 이식성을 확보합니다. 외부에서 n8n 인스턴스에 접근하기 위해 Ngrok을 사용합니다. Ngrok은 로컬에서 실행 중인 서비스를 공개 인터넷에 안전하게 노출하는 터널링 서비스를 제공하며, Always Free 티어에서 고정 IP를 할당받기 어려운 경우 유용하게 활용될 수 있습니다. 다음은 일반적인 설정 흐름입니다.

*   **GCP Compute Engine 인스턴스 생성**: Always Free 자격을 만족하는 f1-micro 인스턴스를 선택하고, Ubuntu 또는 Debian 등 익숙한 OS를 설치합니다.
*   **Docker 및 Docker Compose 설치**: 인스턴스에 Docker와 Docker Compose를 설치하여 n8n 컨테이너 환경을 구축합니다.
*   **n8n Docker Compose 배포**: 위 예시와 유사한 `docker-compose.yml`을 사용하여 n8n을 실행합니다. `volumes` 설정을 통해 n8n 데이터가 인스턴스에 영구적으로 저장되도록 합니다.
*   **Ngrok 설치 및 실행**: Ngrok을 설치하고 `ngrok http 5678` 명령어를 통해 n8n이 실행되는 5678 포트를 외부에 노출합니다. Ngrok에서 제공하는 임시 URL을 통해 n8n 웹 인터페이스에 접근할 수 있습니다.

### Insight
GCP Always Free 티어와 Docker, Ngrok 조합은 n8n 인스턴스를 저비용으로 구축하고 운영하는 효과적인 방법입니다. 특히 Ngrok을 활용하면 고정 IP나 도메인 없이도 외부 접근이 가능하여 초기 테스트 및 학습 단계에서 매우 유용합니다. 하지만 Ngrok의 임시 URL 특성 때문에 장기적인 서비스 운영에는 부적합할 수 있으며, 실제 프로덕션 환경에서는 고정 IP 및 자체 도메인, HTTPS 설정을 고려해야 합니다. n8n의 설정 내보내기/가져오기 기능은 워크플로우의 버전 관리 및 환경 간 이동을 용이하게 하여 개발 효율성을 높이는 데 기여합니다.
