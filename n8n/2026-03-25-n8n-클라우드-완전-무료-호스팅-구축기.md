
### Context

n8n을 로컬 환경이 아닌 클라우드에서 24시간 안정적으로 운영하고자 할 때, 유료 서버 비용은 큰 부담으로 작용할 수 있습니다. 본 문서는 Google Cloud Platform(GCP)의 Always Free 프리 티어 정책을 활용하여 `e2-micro` 인스턴스를 무상으로 확보하고, Docker 및 `docker-compose`를 통해 n8n 환경을 컨테이너화하며, Ngrok을 사용하여 외부 웹훅(Webhook) 경로를 안전하게 노출함으로써 서버 비용 없이 n8n을 클라우드에서 운영하는 방법에 대해 설명합니다. 이 접근 방식은 고정 IP나 복잡한 방화벽, 도메인 설정 없이 빠르고 효율적인 n8n 호스팅 환경을 구축하는 것을 목표로 합니다.

### Core

GCP `e2-micro` 인스턴스에 접속한 후, 다음 `docker-compose.yml` 파일을 생성하고 Ngrok을 설정하여 n8n을 배포할 수 있습니다.

#### n8n Docker Compose 설정

n8n 애플리케이션과 관련 서비스를 컨테이너로 관리하기 위해 `docker-compose.yml` 파일을 작성합니다. 이 설정은 n8n이 데이터를 저장할 볼륨을 정의하여 컨테이너가 재시작되어도 데이터가 유지되도록 합니다.

```yaml
version: '3.8'

services:
  n8n:
    image: n8n.io/n8n
    restart: always
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - N8N_PATH=/
      - N8N_EDITOR_BASE_URL=http://localhost:5678/
      - N8N_BASIC_AUTH_ACTIVE=false # 필요에 따라 true로 변경 및 N8N_BASIC_AUTH_USER, N8N_BASIC_AUTH_PASSWORD 설정
    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:
```

위 파일을 `docker-compose.yml`로 저장한 후, 다음 명령어를 실행하여 n8n을 시작합니다.

```bash
docker-compose up -d
```

#### Ngrok을 이용한 외부 노출

Ngrok은 n8n이 로컬 포트(5678)로 실행될 때, 이를 외부 인터넷에 안전하게 노출하는 터널링 서비스를 제공합니다. GCP 인스턴스에 Ngrok을 설치하고 다음 명령어를 실행하여 n8n 웹훅을 외부에 노출할 수 있습니다.

먼저, Ngrok 계정을 생성하고 인증 토큰(Auth Token)을 설정해야 합니다.

```bash
# Ngrok 설치 (Debian/Ubuntu 기준)
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc \
  | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null && \
  echo "deb https://ngrok-agent.s3.amazonaws.com buster main" \
  | sudo tee /etc/apt/sources.list.d/ngrok.list && \
  sudo apt update && sudo apt install ngrok

# Ngrok 인증 토큰 설정 (Ngrok 대시보드에서 확인)
ngrok config add-authtoken YOUR_NGROK_AUTH_TOKEN

# n8n 포트 5678을 HTTP 터널로 노출
ngrok http 5678
```

위 명령어를 실행하면 Ngrok은 임시 공개 URL을 제공하며, 이 URL을 통해 외부에서 n8n 워크플로우의 웹훅을 호출할 수 있습니다.

### Insight

GCP `e2-micro` 인스턴스와 Docker, Ngrok을 활용한 n8n 무료 호스팅 구축은 서버 운영 비용을 절감하고자 하는 개발자에게 매력적인 대안입니다. GCP의 Always Free 티어는 특정 리소스 사용량까지 무상으로 제공되므로, `e2-micro` 인스턴스를 통해 기본적인 n8n 운영 환경을 무상으로 확보할 수 있습니다.

하지만 이 구성에는 몇 가지 기술적 고려사항이 따릅니다. Ngrok의 무료 티어는 터널링 URL이 임시적(ephemeral)이라는 단점이 있습니다. 이는 Ngrok 터널이 재시작될 때마다 URL이 변경되어, n8n 워크플로우 내에서 외부로 노출된 웹훅 URL을 수동으로 업데이트해야 하는 번거로움이 발생할 수 있습니다. 따라서 장기적인 프로덕션 환경보다는 개발 및 테스트 목적으로 더 적합합니다. 또한 `e2-micro` 인스턴스는 제한된 CPU 및 메모리 리소스를 가지고 있으므로, 복잡하거나 높은 트래픽을 처리하는 n8n 워크플로우에는 성능상의 제약이 있을 수 있습니다.

그럼에도 불구하고, 이 방법은 초기 개발 비용을 최소화하고 n8n의 클라우드 배포 경험을 쌓는 데 매우 유용합니다. Ngrok 대신 Cloudflare Tunnel과 같은 대안을 고려하면 고정 URL 및 추가적인 보안 기능을 확보할 수 있지만, 이는 초기 설정의 복잡성을 증가시킬 수 있습니다.
